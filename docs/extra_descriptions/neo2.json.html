Neo2 layout

Rules for the Neo2 keyboard layout.

Intended to be used with the corresponding keyboard layout file.

Base rules

Base rules will only be active if the corresponding Neo2 keyboard layout is active. This means that the mod3 and mod4 keys will type their actual characters if you activate a different keyboard layout.

Neo2 mod 3 key and layer 4 (Apple keyboards). Rule applied to Apple-made keyboards only.

Toggle mod4 by pressing both mod4 keys simultaneously.

Take this rule if you are using your Mac with a common Apple keyboard.
Neo2 mod 3 key and layer 4 (third party Mac keyboard). Rule applied to all keyboards.

Toggle mod4 by pressing both mod4 keys simultaneously.

Take this rule if you are using your Mac with a non-Apple keyboard that behaves like an Apple keyboard. If either the left mod4 key or the key left of the 1 key are misbehaving, try the "Windows keyboard" rule instead.
Neo2 mod 3 key and layer 4 (Windows keyboard). Rule applied to all non-Apple keyboards.

Toggle mod4 by pressing both mod4 keys simultaneously.

Take this rule if you are using your Mac with a Windows keyboard or with the Apple Magic Keyboard with Numeric Keypad. If either the left mod4 key or the key left of the 1 key are misbehaving, try the "third party Mac keyboard" rule instead.
Neo2 layer 6 Base rule for activating layer 6.

Extension rules

Neo2 mod 4: Map ↖ to Home and ↘︎ to End in terminal apps.

This extra-rule lets you jump to the beginning and the end of a line in your terminal using ↖ and ↘︎ on layer 4. More precisely, it maps mod4+a to Home and mod4+g to End in the following terminal apps: macOS Terminal, iTerm2, Hyper, alacritty, kitty.

Make sure to move this rule above the rule 'Neo2 layer 4' in the Karabiner-Elements preferences.

In case you use the macOS Terminal: You need to perform an additional configuration step in the macOS Terminal preferences: Go to 'Terminal' -> 'Preferences' -> 'Profiles' -> 'Keyboard' and add the following two entries: Key: ↖ Action: \033OH and Key: ↘︎ Action: \033OF.

This rule will only be active if the corresponding Neo2 keyboard layout is active.

Prevent problematic keys (?, /, #, =, and ')') from being treated as option key shortcut

This rule serves as a workaround for issues that layer 3 could cause in some applications. See known issues for more information. The rule maps the problematic keys to a dummy dead key on Numpad 2 + left_option + left_shift followed by the actual key without modifiers.

This rule will only be active if the corresponding Neo2 keyboard layout is active.

This rule serves as a workaround for issues that layer 3 could cause in some applications. See known issues for more information. The rule maps all layer 3 keys to a dummy dead key on Numpad 2 + left_option + left_shift followed by the actual key without modifiers.

This rule will only be active if the corresponding Neo2 keyboard layout is active.
